Electron
Chromium + Node.js 기반 크로스플랫폼 데스크탑 앱. Main/Renderer 두 프로세스 구조.
v28.0.0

Cheerio
서버사이드 HTML 파싱 라이브러리 (jQuery-like API). 실제 사이트 DOM에서 날짜·좌석 데이터 추출.
v1.0.0-rc.12

BrowserView
Electron의 임베드 웹뷰. "원본 보기" 모드에서 실제 사이트를 SlowBro 프레임 안에 렌더링.
Electron built-in

3 시스템 플로우

SlowBro는 두 가지 모드로 동작합니다: SlowBro 변환 모드와 원본 보기 모드

🐢 FlowA — SlowBro 변환 모드
1
URL 입력
주소창에 URL 입력 또는 홈 버튼 클릭
2
사이트 감지
URL 패턴으로 giants / interpark / korail 등 식별
3
HTML Fetch
http.get 또는 mock 파일 읽기
4
Cheerio 파싱
날짜·좌석·제출버튼 셀렉터 추출 → Steps 배열 생성
5
테마 반환
사이트 색상 팔레트를 함께 반환
6
단순화 UI
원본 사이트 색상 유지 + 광고·팝업 제거 + 큰 글씨 4단계 플로우
🌐 FlowB — 원본 보기 모드
1
모드 전환
"원본 보기" 탭 클릭
2
BrowserView 생성
Main 프로세스에서 BrowserView 인스턴스 생성
3
Bounds 계산
navbar.bottom ~ window.innerHeight 로 좌표 계산
4
실제 사이트 렌더링
Chromium이 그대로 렌더링. JS/CSS 완전 동작
✅ FlowC — 예매 단계 (Guard 포함)
사용자 클릭 ↓ guard.checkClick(sessionId, timestamp) ↓ 클릭 간격 < 250ms 이고 3회 이상? ├── YES → 경고 표시 + 쿨다운 (매크로 차단) └── NO → 다음 스텝으로 진행 ↓ 최종 제출 단계 ↓ guard.checkSession(userId, eventId) ↓ 동일 유저+이벤트 시도 횟수 ≥ 1? ├── YES → 중복 예매 차단 └── NO → 예매 완료 + TTS 안내

5 IPC API 명세

invoke = 양방향 (응답 있음)   send = 단방향 (응답 없음)   event = Main → Renderer 푸시

모든 API는 window.slowbro.* 로 Renderer에서 접근. contextBridge를 통해 안전하게 노출됨.
